# 课件学习指南监修标准

本文档沉淀 AI 课程 PPT 学习指南的通用监修标准，适用于后续所有课件学习指南。单个 PPT 指南正文应专注课程内容，不承载本类通用规范或监修日志。

## 核心原则

1. **名词不裸奔**：典故、算法名、术语首次出现时，要交代场景、要解决的问题，以及它在本章中的作用。
2. **专业词首次白话解释**：先验、似然、基率、稳态、阻尼因子、隶属度等术语，第一次出现要先翻成普通话；能讲清的地方不要堆黑话。
3. **抽象困难要拆机制**：遇到「条件概率爆炸」「稳态」「梯度消失」这类抽象难点，要拆成可追踪机制。例如说明证据组合数如何变多、概率表项为何快速增长，或反复迭代后分布如何不再变化。
4. **例子必须服务理解**：石油、农夫、TALL、马尔可夫两状态等例子，要明确说明它在解释哪个公式或概念；有课件或 raw 确认的数值要保留关键计算，没有来源的数值不硬编。
5. **先建概念再给公式**：先讲问题背景、直觉框架和变量含义，再给公式表。公式出现后要解释每一项含义，以及它在解题流程中的位置。
6. **重点公式表不可被摘要吃掉**：CF 等期末手算重点必须保留符号表、适用条件、手算顺序、公式表和例题步骤，不能只写成概念摘要。
7. **备考权重明确**：区分必考手算、概念理解、选择题层级和课堂补充内容，避免把所有板块写成同等优先级。
8. **格式风险检查**：定稿前检查标题层级、表格列数、公式块、mermaid 代码块、引用路径、术语中英文格式和 Markdown 分隔线，避免因格式问题影响阅读或 PDF 导出。

## 监修检查清单

- 每个大板块是否先回答「本节要解决什么问题」。
- 首次出现的核心术语是否有中文、英文和白话解释。
- 抽象概念是否有机制拆解，而不是只给结论。
- 例子是否紧贴概念或公式，且没有无来源数值。
- 重点公式、符号表、适用条件和手算模板是否完整。
- 复习优先级是否与课堂和期末定位一致。
- 易混概念是否通过对比表或提醒块区分。
- Markdown 表格、公式、mermaid、路径引用是否能正常渲染。
