根据**课件 05 Uncertainty**与 **Week 13 记录**，确定性因子（Certainty Factor, CF）模型主要用于处理规则系统中的不确定性，包括证据的不确定性、规则本身的不确定性以及多条规则指向同一结论时的不确定性合成 [1, 2]。

以下是一个完整的数值手算例题，展示了从证据合成到多条规则合成的计算全过程。

---

### **确定性因子（CF）合成数值手算例题**

**已知条件：**
*   **规则 1 (r1):** IF $E_1$ **AND** $E_2$ THEN $H$ ($CF(H, E_{r1}) = 0.8$) [3, 4]
*   **规则 2 (r2):** IF $E_3$ THEN $H$ ($CF(H, E_{r2}) = 0.6$) [4, 5]
*   **当前观测到的证据确定性：**
    *   $CF(E_1, e) = 0.5$
    *   $CF(E_2, e) = 0.9$
    *   $CF(E_3, e) = 0.4$

**计算目标：** 计算在已知证据 $e$ 下，结论 $H$ 的综合确定性因子 $CF_{comb}(H)$。

---

#### **第一步：计算规则 1 前提条件的确定性 $CF(E_{r1}, e)$**
当规则前提由多个证据通过逻辑关系（如 **AND**）组合时，需先进行证据合成。
*   **使用公式：** $CF(E_1 \text{ AND } E_2, e) = \min[CF(E_1, e), CF(E_2, e)]$ [4, 6]
*   **计算过程：**
    $$CF(E_{r1}, e) = \min[0.5, 0.9] = 0.5$$
*   **说明：** 规则 1 的前提条件受限于最弱的一个证据确定性。

#### **第二步：计算规则 1 推导出的结论确定性 $CF_1(H, e)$**
根据前提条件的确定性与规则自身的确定性，计算该规则对结论的贡献。
*   **使用公式：** $CF(H, e) = CF(E, e) \times CF(H, E)$ [4, 7]
*   **计算过程：**
    $$CF_1(H, e) = 0.5 \times 0.8 = 0.4$$
*   **说明：** 规则 1 为结论 $H$ 提供了 **0.4** 的确定性支持。

#### **第三步：计算规则 2 推导出的结论确定性 $CF_2(H, e)$**
对第二条独立规则进行同样的计算。
*   **使用公式：** $CF(H, e) = CF(E, e) \times CF(H, E)$ [4, 7]
*   **计算过程：**
    $$CF_2(H, e) = CF(E_3, e) \times CF(H, E_{r2}) = 0.4 \times 0.6 = 0.24$$
*   **说明：** 规则 2 为结论 $H$ 提供了 **0.24** 的确定性支持。

#### **第四步：合成两条规则的确定性因子 $CF_{comb}$**
当有两条规则支持同一个结论 $H$ 时，需要将它们的结果进行合成。
*   **判定条件：** 由于 $CF_1(H, e) = 0.4 > 0$ 且 $CF_2(H, e) = 0.24 > 0$，属于“同号且为正”的情况 [5]。
*   **使用公式：** $CF_{comb}(CF_1, CF_2) = CF_1 + CF_2(1 - CF_1)$ [5]
*   **计算过程：**
    $$CF_{comb}(H) = 0.4 + 0.24 \times (1 - 0.4)$$
    $$CF_{comb}(H) = 0.4 + 0.24 \times 0.6$$
    $$CF_{comb}(H) = 0.4 + 0.144 = 0.544$$
*   **说明：** 最终合成的确定性因子为 **0.544**。这体现了确定性因子模型“证据累积”的特性：多个独立证据的支持会使结论的确定性进一步增强 [2]。

---

### **总结公式来源**
1.  **证据合成（AND）：** 取最小值 $\min$ [6]。
2.  **结论传递：** 前提确定性 $\times$ 规则确定性 [4]。
3.  **规则合成（CFCOMB）：** 对于同为正值的 $CF_1, CF_2$，采用 $CF_1 + CF_2(1 - CF_1)$ 进行增强合成 [5]。
