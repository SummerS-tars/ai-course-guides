根据 Week 8 关于**生成模型**（Generative Models）的课程内容及相关教材，以下是 5 组最易混淆的核心概念对比说明：

### 1. ELBO (证据下界) vs. KL 散度 (Kullback-Leibler Divergence)
在变分自编码器（VAE）的数学推导中，这两者是分解对数似然 $\log P(X)$ 的关键组成部分。

| 特性 | ELBO (证据下界) | KL 散度 ($D_{KL}$) |
| :--- | :--- | :--- |
| **定义** | 数据对数似然的一个可处理的下界 [1]。 | 衡量两个概率分布之间差异的非负度量 [1, 2]。 |
| **在 VAE 中的作用** | 作为**优化目标**。通过最大化 ELBO 来间接最大化难以直接处理的对数似然 [1]。 | 作为**正则化项**。强制变分后验 $Q(Z\|X)$ 接近预设的先验 $P(Z)$（通常为标准正态分布），以约束潜在空间 [3]。 |
| **物理意义** | 包含“重构项”和“正则项”，平衡重建质量与隐空间的紧致性 [3]。 | 确保隐空间的**连续性和平滑性**，防止模型通过为每个样本分配隔离的点来寻找“捷径” [4]。 |
| **关系公式** | $\log P(X) = \text{ELBO} + D_{KL}(Q_\phi(Z\|X) \parallel P(Z\|X))$ [1]。 | |

### 2. VAE (变分自编码器) vs. 扩散模型 (Diffusion Model)
两者都属于生成模型，但生成的步长和逻辑存在本质区别。

| 特性 | 变分自编码器 (VAE) | 扩散模型 (Diffusion Model) |
| :--- | :--- | :--- |
| **生成步长** | **一步生成**（一步到位）。直接从潜在空间 $Z$ 解码生成完整的、高分辨率的图像 [5, 6]。 | **多步生成**（分而治之）。将生成过程分解为多个简单的、逐步去噪的步骤 [6]。 |
| **核心思路** | 通过编码器映射隐空间分布，再由解码器还原 [5]。 | 通过“概率路径”：反向逐步加噪至纯噪声，正向学习每一步的去噪过程以恢复图像 [5, 6]。 |
| **局限与优势** | 生成复杂高分辨率图像难度大，图像往往倾向于模糊 [6, 7]。 | 能够生成质量更高、更真实的图像，显著降低了学习难度 [6]。 |
| **理论关联** | 扩散模型的每一步转换都可以看作一个简单的 VAE [5]。 | 扩散模型可视为 VAE 的一种层次化、多步化扩展形式 [6]。 |

### 3. 扩散模型 (Diffusion) vs. 流匹配 (Flow Matching)
流匹配是近年来在扩散模型基础上演进出的更直接的框架。

| 特性 | 扩散模型 (Diffusion) | 流匹配 (Flow Matching) |
| :--- | :--- | :--- |
| **数学模型** | 基于**随机微分方程 (SDE)**。每一步都涉及随机噪声的添加与去除 [5, 6]。 | 基于**常微分方程 (ODE)**。学习一个确定的向量场（速度场）指明移动方向 [5, 6]。 |
| **确定性** | 具有随机性。由于每一步都有随机采样，多样性丰富 [6]。 | 具有确定性。从噪声到数据的路径是直接且确定的 [6]。 |
| **性能表现** | 生成结果的创造性较强 [6]。 | 训练目标更直接，收敛更快，在需要高保真度的领域（如医学图像）更有优势 [6]。 |
| **三者统一** | 流匹配上加入随机因素即得扩散模型 [8]。 | 将步数设为 1 即退化为 VAE [8]。 |

### 4. GAN (生成对抗网络) vs. VAE (变分自编码器)
这是两类经典的、采用不同哲学思想训练的生成模型。

| 特性 | 生成对抗网络 (GAN) | 变分自编码器 (VAE) |
| :--- | :--- | :--- |
| **训练机制** | **对抗博弈**。生成器与判别器通过零和游戏相互对抗、共同进化 [9-11]。 | **变分推断**。通过最大化似然下界（ELBO）使模型分布逼近真实分布 [1, 12]。 |
| **模型类型** | **隐式模型**（Implicit Model）。可以生成样本，但无法直接给出样本的概率 [13]。 | **显式概率模型**。旨在学习联合分布 $P(X, Z)$ [1]。 |
| **生成质量** | 图像非常自然、逼真，能生成高清的人像等 [11, 14]。 | 样本往往比较**模糊** [7]。 |
| **训练难度** | 难以稳定训练（收敛性问题），对超参数非常敏感 [15, 16]。 | 训练过程相对稳定且容易实现 [7]。 |

### 5. 对抗样本 (Adversarial Examples) vs. 对抗训练 (Adversarial Training)
这两者在鲁棒性研究中经常被提及，一个代表“攻击”，一个代表“防御”。

| 特性 | 对抗样本 (Adversarial Examples) | 对抗训练 (Adversarial Training) |
| :--- | :--- | :--- |
| **本质** | **攻击性输入**。通过在正常输入中加入微小的、肉眼难察的扰动，使模型产生错误判断 [17, 18]。 | **防御性手段/正则化**。在训练过程中主动加入对抗扰动样本，提高模型的鲁棒性 [19]。 |
| **成因** | 神经网络的过度线性（ excessive linearity）使得小扰动在多层传递中被放大 [19, 20]。 | 旨在消除模型对输入的极度敏感性，鼓励模型在训练数据邻域内保持局部常数性 [20]。 |
| **目的** | 揭示模型在安全和鲁棒性上的漏洞 [21]。 | 降低测试集上的泛化误差，赋予模型抵抗对抗性扰动的能力 [19]。 |

这些概念构成了 Week 8 关于深度生成模型及其安全性讨论的核心脉络。
