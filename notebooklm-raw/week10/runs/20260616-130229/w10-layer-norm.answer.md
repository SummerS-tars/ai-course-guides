根据 **Week 10 课程记录**、**课件 09** 以及参考书相关章节，现对 Layer Normalization (LN) 与 Batch Normalization (BN) 的区别进行对比，并解释 Transformer 采用 LN 的原因。

### Layer Norm (LN) 与 Batch Norm (BN) 区别对比表

| 比较维度 | Batch Normalization (BN) | Layer Normalization (LN) |
| :--- | :--- | :--- |
| **归一化对象** | 在一个 **Mini-batch** 内，对所有样本的同一个特征维度进行归一化 [1, 2]。 | 对 **单个样本** 内的所有特征维度进行归一化 [3]。 |
| **计算统计量** | 计算 Mini-batch 的均值 $\mu$ 和方差 $\sigma$ [1, 4]。 | 计算单个样本所有隐藏单元的均值和方差 [3, 5]。 |
| **Batch Size 依赖** | 高度依赖 Batch Size。Batch 过小时，统计量估计不准，训练会不稳定 [6, 7]。 | **不依赖 Batch Size**。归一化操作在单个样本内部完成 [3]。 |
| **序列长度敏感性** | 对变长序列处理困难，因为不同时间步的统计特性可能差异巨大 [8, 9]。 | 对变长序列非常友好，每个位置的归一化是独立的 [3, 8]。 |
| **主要应用领域** | 广泛应用于 **卷积神经网络 (CNN)** 和计算机视觉任务 [2, 6, 10]。 | 广泛应用于 **循环神经网络 (RNN)** 和 **Transformer** 等 NLP 任务 [3, 11]。 |
| **核心目的** | 解决“内部协变量偏移” (Internal Covariate Shift) [2, 12]。 | 稳定深层网络或序列模型的梯度流 [3, 13]。 |

### 为何 Transformer 采用 Layer Norm (LN)？

根据课程记录与参考资料，Transformer 架构选择 LN 而非 BN，主要基于以下深层原因：

#### 1. 变长序列的处理需求
在自然语言处理中，输入句子的长度通常是不固定的 [8, 9]。
*   **BN 的局限性**：BN 依赖于 Batch 内的统计特性。在处理变长序列时，不同样本在同一位置（时间步）可能存在填充（Padding）或是完全不同的语义信息，导致跨样本计算的均值和方差失去统计意义 [2, 8]。
*   **LN 的优势**：LN 是在单个样本的所有特征维度上进行归一化。无论句子长短，LN 仅针对当前 token 的隐藏向量进行操作，保证了归一化的一致性和稳定性 [3, 14]。

#### 2. 对 Mini-batch 大小的鲁棒性
*   **Transformer 的高开销**：Transformer 模型参数量巨大（如 GPT、BERT 等），通常对显存需求极高 [15]。由于硬件资源限制（如 4090/5090 显存），训练时往往只能使用较小的 Batch Size [15, 16]。
*   **BN 的风险**：正如课程所强调，BN 在小 Batch 下表现极差，因为统计量的方差会变得非常大，导致模型难以收敛 [6, 7]。
*   **LN 的稳定性**：LN 独立于 Batch Size。即使 Batch Size 为 1，LN 依然可以正常计算并提供稳定的归一化效果，这对于超大模型的训练至关重要 [3, 17]。

#### 3. 预测阶段的一致性
*   **BN 的复杂性**：BN 在推理阶段需要使用训练期间累积的滚动平均（Running Averages）值，这增加了模型部署的复杂性 [17, 18]。
*   **LN 的简洁性**：LN 在训练和推理时的行为完全一致，不需要存储额外的全局统计量，更符合 Transformer 这种追求高度并行化和计算效率的架构要求 [3, 14]。

#### 4. 优化景观的改善
*   **改善条件数**：Week 10 课程提到，归一化的核心目的是改善损失函数景观（Loss Landscape）的几何性质，降低海森矩阵的条件数 [2, 19]。对于 Transformer 这种包含多头注意力（Multi-head Attention）和深度堆叠的结构，LN 能更有效地防止梯度消失或爆炸，使训练曲面更平滑 [2, 20]。
