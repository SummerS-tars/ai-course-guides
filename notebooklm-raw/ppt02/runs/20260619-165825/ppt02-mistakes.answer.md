在“课件 02 知识表示”及相关参考资料中，以下是 4 组最易混淆的核心概念对比说明：

### 1. 专家系统 (Expert System) vs. 传统程序 (Traditional Program)
这是课件开篇提出的首要对比，旨在区分人工智能系统与一般软件的本质差异 [1]。

| 特性 | 传统程序 | 专家系统 |
| :--- | :--- | :--- |
| **核心公式** | **算法 + 数据结构** (Algorithms + Data Structures) [1] | **知识 + 推理** (Knowledge + Inference) [1] |
| **处理能力** | 侧重于数值计算和确定的过程性处理 [2] | 侧重于定性推理和处理不确定性信息 [2] |
| **灵活性** | 逻辑硬编码在程序中，难以修改 [3] | 知识与控制分离，易于更新知识库而不影响推理机 [4] |
| **可解释性** | 通常是“黑箱”过程，无法解释运行逻辑 | 具有解释机制，可回答“为什么”和“如何” [5, 6] |

### 2. 知识表示 (KR) vs. 数据库 (Database)
两者虽然都涉及信息存储，但在目的和推理能力上有显著区别 [7, 8]。

| 特性 | 数据库 (Database) | 知识表示 (KR) |
| :--- | :--- | :--- |
| **数据本质** | 存储显式事实，通常采用固定模式（如表结构） [8, 9] | 表示关于世界的陈述、一般性规则及复杂逻辑关联 [8, 10] |
| **推理机制** | 缺乏从已有事实自动推导新事实的通用机制 [7] | 核心目标是通过推理机推导出隐含的结论 [11, 12] |
| **不完整性处理** | 难以处理“未知”或析取（Or）等不完整信息 [8] | 能够表达否定、不确定性和部分已知的信息 [13] |
| **语义关联** | 关系通常由外部程序逻辑解释 | 语义结构（如分类层级）是表示法本身的一部分 [12, 14] |

### 3. 语义网络 vs. 框架 vs. 概念图
这三者是符号主义中最为相近的结构化表示方法，但在结构能力上层层递进 [12]。

| 方法 | 核心结构 | 独特能力 / 限制 |
| :--- | :--- | :--- |
| **语义网络** | 节点表示对象/概念，有向弧表示关系 [12, 15] | 直观、易于实现继承推理；但难以处理三元以上关系，且存在语义歧义 [12] |
| **框架 (Frame)** | 具有命名“槽”(Slots) 和“值”(Values) 的结构化对象 [12, 16] | 类似于面向对象中的“类”，支持**嵌套**、**默认值**和**程序挂钩**（执行特定副作用） [12, 17] |
| **概念图** | 二部图，节点分为“概念节点”和“关系节点” [12, 18] | 改进了语义网络，通过关系节点连接多个概念，能**直接表示多元关系**（如“谁在何时何地做了什么”） [12, 19] |

### 4. 继承 (Inheritance) vs. 默认 (Default)
这两个概念常在层级结构中结合使用，但其逻辑含义不同 [20]。

| 概念 | 定义与推理方向 | 冲突处理 |
| :--- | :--- | :--- |
| **继承 (Inheritance)** | 子类或实例自动获得父类定义的属性 [21, 22] | 通常是严格的逻辑推导（如：猫是哺乳动物 $\rightarrow$ 猫有皮毛） [21] |
| **默认 (Default)** | 在缺乏明确证据时，假定某属性值为某个特定值 [17, 20] | 具有**非单调性**：当发现具体事实与默认值冲突时，默认值会被覆盖（Override） [20, 23] |
| **典型示例** | “金丝雀”继承“鸟”的“有羽毛”属性 [22] | “鸟”默认会飞，但对于特定的“鸵鸟”，该默认值被“不会飞”的事实覆盖 [20, 24] |
