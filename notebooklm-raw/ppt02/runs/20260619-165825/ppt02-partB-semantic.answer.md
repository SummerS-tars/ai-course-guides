根据**课件 02 (Knowledge Representation)** 及 **Week 12** 的课程内容，语义网络（Semantic Network）通过图形化的方式组织知识，其中著名的“金丝雀分层图”揭示了人类如何高效地存储和检索信息 [1-3]。

### 1. 节点与有向边的含义

在语义网络中，知识被表示为由节点和弧（有向边）组成的图 [4]。

*   **节点 (Nodes)**：代表**对象（Objects）**或**类别（Categories/Concepts）** [1, 3]。
    *   例如：在图中，“动物 (ANIMAL)”、“鸟 (BIRD)”、“金丝雀 (CANARY)” 都是节点 [1]。
*   **有向边 (Directed Edges/Links)**：代表节点之间的**关系 (Relationships/Associations)** [4]。
    *   **is-a 关系**：表示成员关系（Instance of）或子类关系（Subset of） [1, 5, 6]。它构建了语义网络的层级结构（Taxonomic Hierarchy）。例如，“金丝雀 `is-a` 鸟”意味着金丝雀是鸟类的一个子集 [1]。
    *   **属性关系 (Property Links)**：标注为 `can` (能)、`has` (有)、`is` (是) 等，用于描述类别的特征 [1, 4, 6]。
        *   例如：鸟 `has` 翅膀 (Wings)、鸟 `can` 飞行 (Fly) [1]。

### 2. 继承 (Inheritance) 的推理机制

继承机制允许信息存储在分类层级中**最合适且最高**的抽象层级上，从而减少冗余并保证一致性 [7, 8]。

*   **推理过程**：当需要查询某个对象的属性（如“金丝雀会飞吗？”）时，推理算法会从该对象节点（CANARY）开始，检查其本地是否有该属性 [9, 10]。
*   **向上搜索**：如果本地没有找到，算法会沿着 `is-a` 链接**向上追溯**到父节点（BIRD），并在父节点处查找属性 [1, 9, 10]。
*   **属性继承**：子类会自动继承父类及其所有祖先节点的属性 [4, 8]。除非在更具体的层级上有明确的冲突说明（即“异常处理”，如鸵鸟 `cannot` 飞行），否则推理会持续向上直到根节点 [6]。

### 3. 金丝雀→鸟→动物层级的推理链例子

**推理问题**：“金丝雀 (Canary) 有皮肤 (Skin) 吗？”

**推理链步骤**：
1.  **起点**：定位到 **CANARY** 节点。本地属性只有“能歌唱 (Sing)”和“是黄色 (Yellow)”，未发现“皮肤”属性 [1]。
2.  **第一级追溯**：沿着 `is-a` 边移动到 **BIRD** 节点。这里定义了“有翅膀”、“能飞”和“有羽毛”，仍未发现“皮肤” [1, 9]。
3.  **第二级追溯**：继续沿着 `is-a` 边移动到 **ANIMAL** 节点 [1, 9]。
4.  **发现属性**：在 **ANIMAL** 节点处，系统发现标注有 `has` **Skin**（有皮肤）的属性链接 [1]。
5.  **结论**：由于金丝雀是一种鸟，鸟是一种动物，因此**金丝雀继承了动物的所有属性**。推理结果：金丝雀有皮肤 [1, 9]。

该图展示了一个基于心理学实验证据（反应时间差异）构建的知识层级模型：

*   **顶层（ANIMAL 动物）**：最抽象的层级。定义了所有动物共有的基本生存属性，如：`can` **Breathe** (能呼吸)、`can` **Move** (能运动)、`has` **Skin** (有皮肤) [1]。
*   **中间层（BIRD 鸟 / FISH 鱼）**：
    *   **BIRD**：通过 `is-a` 链接指向动物。特有属性包括：`has` **Wings** (有翅膀)、`can` **Fly** (能飞)、`has` **Feathers** (有羽毛) [1]。
*   **底层（特定物种，如 CANARY 金丝雀 / OSTRICH 鸵鸟）**：
    *   **CANARY**：通过 `is-a` 链接指向鸟。特有属性为：`can` **Sing** (能歌唱)、`is` **Yellow** (是黄色) [1]。
    *   **OSTRICH**：通过 `is-a` 链接指向鸟。具有特有属性：`is` **Tall** (高大的)；同时包含一个**异常覆盖 (Overriding)** 属性：`cannot` **Fly** (不能飞) [1, 6]。
