在人工智能（H）课程第 12 周的内容中，围绕 **Transformer 架构**与**生成模型**有几组核心概念极易混淆。以下是为您梳理的 5 组最易混淆概念的对比表：

### 1. 编码器 (Encoder) vs 解码器 (Decoder)
这两者是 Transformer 架构的两大支柱，主要区别在于对上下文信息的处理方式。

| 维度 | 编码器 (Encoder) | 解码器 (Decoder) |
| :--- | :--- | :--- |
| **注意力类型** | **双向注意力**：每个位置可同时融合上文和下文信息 [1]。 | **掩码/因果注意力**：受时序约束，只能看到已生成的部分（单向） [1]。 |
| **核心组件** | 包含多头注意力子层和前馈网络子层 [1]。 | 在编码器基础上增加了**交叉注意力**子层，用于融合输入序列信息 [1]。 |
| **主要功能** | 将输入序列转换为高层特征表示（提取特征） [1]。 | 根据编码信息和已生成内容，逐步生成目标序列 [1]。 |
| **应用代表** | **BERT** [1] | **GPT** [1] |

---

### 2. 自注意力 (Self-Attention) vs 交叉注意力 (Cross-Attention)
这两者决定了模型“检索”信息的来源范围。

| 维度 | 自注意力 (Self-Attention) | 交叉注意力 (Cross-Attention) |
| :--- | :--- | :--- |
| **向量来源** | **Q, K, V 均来自同一个序列** [1, 2]。 | **Q 来自当前序列（解码器）**，**K, V 来自另一个序列（编码器）** [1, 3]。 |
| **存在位置** | 编码器所有层、解码器的首个注意力子层 [1]。 | 仅存在于解码器的模块中（通常是第二个注意力子层） [1]。 |
| **主要目的** | 捕捉同一序列内不同位置元素间的依赖关系 [4, 5]。 | 实现不同序列间的信息融合（如翻译中原句与目标句的对应） [1, 3]。 |

---

### 3. BERT vs GPT (预训练范式)
这是大语言模型发展史上的两条分水岭，目前 GPT 范式已成为主流 [6]。

| 维度 | BERT (双向编码器表示) | GPT (生成式预训练) |
| :--- | :--- | :--- |
| **架构** | **仅训练编码器 (Encoder-only)** [1, 7]。 | **仅训练解码器 (Decoder-only)** [1, 8]。 |
| **信息可见性** | 双向可见，通过掩码单词进行预训练 [1, 7]。 | 仅单向可见（因果），给定上文预测下文 [1, 9]。 |
| **擅长任务** | 文本理解类（如分类、标注、信息抽取） [1, 10]。 | 文本生成类，具备强大的**零样本 (Zero-shot)** 泛化能力 [1, 11]。 |
| **计算特性** | 推理时通常需全量计算。 | 支持 Q/K/V 缓存复用，生成速度极快 [1]。 |

---

### 4. VAE vs 扩散模型 vs 流匹配 (图像生成)
这三者在理论上具有统一性，但生成过程的深度不同。

| 维度 | 变分自编码器 (VAE) | 扩散模型 (Diffusion Model) | 流匹配 (Flow Matching) |
| :--- | :--- | :--- | :--- |
| **生成机制** | **一步生成**：从简单隐分布直接映射到真实分布 [6]。 | **多步生成**：逐步去噪，每一步可视为一个简单的 VAE [6]。 | **向量场演化**：通过学习“速度场”指明数据移动方向 [6]。 |
| **数学本质** | 证据下界 (ELBO) 优化 [12]。 | 随机微分方程 (SDE) 框架下的去噪拟合 [6]。 | 常微分方程 (ODE) 框架下的轨迹拟合 [6]。 |
| **三者关系** | **流匹配是通用框架**；在此基础上加入随机因素即为**扩散模型**；将时间步设为 1 即为 **VAE** [13]。 | | |

---

### 5. 因果注意力 (Causal Attn) vs 双向注意力 (Bidirectional Attn)
这是处理序列数据时最基本的逻辑差异。

| 维度 | 因果注意力 (Causal/Masked) | 双向注意力 (Bidirectional) |
| :--- | :--- | :--- |
| **可见范围** | 每个位置只能“看到”它之前的元素 [1]。 | 每个位置能“看到”整个序列的所有元素 [1]。 |
| **应用场景** | **预测与生成**。必须符合时间先后顺序，不能预测未来 [1, 14]。 | **分析与理解**。当输入序列已完全给定时使用 [1]。 |
| **模型范型** | GPT 系列、Transformer 解码器 [1]。 | BERT、Transformer 编码器 [1]。 |
