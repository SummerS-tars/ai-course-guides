根据**课件 09**、**Week 10 课程记录**以及参考书相关章节，神经网络的优化与泛化技术是确保深层模型能够收敛并具备泛化能力的核心。以下是针对您提及的三个方面的详细讲解与对比。

### 1. 参数优化算法的演进：SGD → Momentum → Adam

优化算法的发展脉络体现了从“固定步长”到“引入惯性”，再到“自适应调整步长”的过程 [1]。

*   **随机梯度下降 (SGD)**：最基础的优化器。在小批量（Mini-batch）训练中，它通过计算一小部分样本的梯度来更新参数 $\theta$ [1]。虽然计算开销小且具有一定的随机性（有助于逃离鞍点），但在面临“峡谷”状的损失曲面时容易产生震荡，收敛缓慢 [1, 2]。
*   **动量法 (Momentum)**：引入了物理学中的惯性概念。它通过**指数加权平均**跟踪梯度的历史趋势，更新公式中包含上一步的速度项 [1]。当梯度方向一致时加速收敛，方向不一致时抵消震荡，从而帮助参数跨越平坦区域 [1, 3]。
*   **Adam (Adaptive Moments)**：目前最常用的优化器，结合了 **Momentum**（一阶矩估计）和 **RMSprop**（二阶矩估计，自适应学习率）[4]。它同时跟踪梯度的均值和未中心化的方差，并进行偏差修正，以消除训练初期的统计偏差 [5]。

#### 优化算法对比表
| 特性 | SGD | Momentum | Adam |
| :--- | :--- | :--- | :--- |
| **核心机制** | 直接沿梯度反方向更新 [6] | 累积历史梯度方向（动量/惯性）[7] | 结合动量与自适应学习率（一阶/二阶矩）[5] |
| **优势** | 简单，是性能衡量的基线 [1] | 抑制震荡，加速经过狭长地带 [3] | 自动调整每个参数的学习率，鲁棒性强 [4, 5] |
| **缺点** | 容易困在局部极小值或产生震荡 [2] | 引入了额外的超参数（动量系数）[8] | 对某些任务可能不如调优后的 SGD 泛化好 [9] |
| **适用场景** | 简单模型或海量数据 [10] | 需要更快收敛速度的深层网络 [11] | 复杂深度网络，如 Transformer [4] |

### 2. 初始化与归一化：Xavier/He 与 BN/LN

参数初始化的目的是破坏对称性（避免神经元同步更新）并保持信号强度（方差稳定）；归一化则是通过调整分布来降低优化难度 [12]。

*   **Xavier 初始化**：适用于 **Sigmoid** 和 **Tanh** 激活函数。其核心思想是使每一层输出的方差与输入一致 [13]。
*   **He 初始化 (Kaiming Initialization)**：专为 **ReLU** 激活函数设计。由于 ReLU 会将一半输入置零（方差减半），He 初始化将权重方差设为 $2/n_{in}$ 以进行补偿 [13]。
*   **批量归一化 (Batch Norm, BN)**：对每个小批量数据的内部层激活值进行标准化（零均值、单位方差）[14]。BN 通常置于线性变换之后、激活函数之前，因为它利用了线性输出更符合高斯分布的特性 [12]。
*   **层归一化 (Layer Norm, LN)**：在 Transformer 架构中更为常见。它在单个样本的所有特征维度上计算均值和方差，而不是在 Batch 维度 [15]。

#### 初始化与归一化对比表
| 类别 | 技术项 | 核心目标 / 适用场景 | 备注 |
| :--- | :--- | :--- | :--- |
| **初始化** | **Xavier** | 配合 Sigmoid/Tanh，保持各层方差稳定 [13] | 基于线性假设推导 [16] |
| | **He** | 配合 ReLU，补偿因零截断导致的方差损失 [13] | 目前 ReLU 网络的默认选择 [13, 17] |
| **归一化** | **Batch Norm** | 解决内部协变量偏移，加速收敛，具正则化效果 [12, 18] | 依赖 Batch 统计量，受 Batch Size 影响 [1, 19] |
| | **Layer Norm** | 适用于变长序列（如 RNN/Transformer）[15] | 在单个样本内独立计算 [15] |

### 3. 正则化方法：L2、Dropout、早停

正则化旨在通过限制模型复杂度来防止**过拟合**，从而提高泛化能力 [13, 20]。

*   **L2 正则化 (权重衰减)**：在损失函数中增加权重平方项，惩罚过大的参数值 [13, 21]。它倾向于使权重变得“均匀且微小”，物理意义上相当于减小了模型的曲率（复杂度）[13]。**偏置项通常不参与正则化** [13]。
*   **随机失活 (Dropout)**：训练时以概率 $p$ 随机将神经元输出置零 [22]。这相当于在每次迭代中训练一个不同的“瘦”网络，最终效果类似于集成学习（Bagging）[23, 24]。为保持测试时的一致性，通常使用 **Inverted Dropout**（训练时除以 $1-p$ 进行缩放）[22]。
*   **早停 (Early Stopping)**：监控验证集损失，一旦损失长时间不再下降（超过预设的“耐心度”）则停止训练 [22, 25]。这被认为是最简单有效的正则化手段，因为它限制了模型能够利用训练数据的迭代次数，在二次代价函数下其效果等价于 L2 正则化 [22, 26]。

#### 正则化技术对比表
| 方法 | 操作方式 | 防过拟合原理 | 关键点 |
| :--- | :--- | :--- | :--- |
| **L2 正则化** | 损失函数 + 权重平方项 [13] | 惩罚高参数值，降低模型复杂度 [27] | 权重衰减系数 $\lambda$ 是关键超参数 [28] |
| **Dropout** | 随机屏蔽部分神经元 [23] | 减少神经元间的协同适应，类似集成学习 [29, 30] | 仅在训练阶段启用，测试时全开启 [22, 24] |
| **早停** | 在验证集性能拐点停止 [31] | 限制有效容量，防止模型过度拟合噪声 [32] | 简单易行，被广泛推荐 [25, 33] |
