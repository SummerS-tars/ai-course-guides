根据 Week 7 的课程笔记 [1, 2] 及相关参考书 [3, 4]，以下是卷积神经网络（CNN）中最易混淆的 5 组核心概念对比表：

### Week 7 核心概念易混淆点对比表

| 概念组 | 维度 | 概念 A：**卷积 (Convolution)** | 概念 B：**互相关 (Cross-correlation)** | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **数学定义与实现** | **操作细节** | 在数学严格定义中，卷积核在与输入进行内积运算前需要进行**翻转（Flip）** [1]。 | 在深度学习实现中，通常**直接对应位置相乘求和**，不进行核翻转 [5, 6]。 | [1, 5, 6] |
| | **应用背景** | 源于信号处理，旨在满足交换律和结合律 [1]。 | 深度学习中的“卷积层”实际上多为互相关，因为模型会自动学习翻转后的权重 [5, 7]。 | [1, 5, 7] |

| 概念组 | 维度 | 概念 A：**通道 (Channel)** | 概念 B：**特征图 (Feature Map)** | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **数据表征含义** | **定义** | 指数据的深度维度。如 RGB 图像有 3 个输入通道 [8, 9]。 | 卷积层的输出结果，反映了输入中特定特征（如边缘）的分布密度 [10, 11]。 | [8-11] |
| | **数量关系** | **卷积核的通道数**必须与**输入特征图的通道数**一致 [8]。 | **输出特征图的通道数**（深度）等于该层所使用的**卷积核（Filter）的数量** [8, 10]。 | [8, 10] |

| 概念组 | 维度 | 概念 A：**平移等变性 (Equivariance)** | 概念 B：**平移不变性 (Invariance)** | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **平移鲁棒性** | **核心特性** | 输入物体移动了多少，输出特征图中的响应也随之移动多少 [2, 12]。 | 输入物体发生微小移动，输出结果（通常是池化后）保持不变 [2, 13]。 | [2, 12, 13] |
| | **所属层级** | 主要由**卷积层**（参数共享）提供 [2, 14]。 | 主要由**池化层**（特别是最大池化）提供 [2, 15]。 | [2, 14, 15] |

| 概念组 | 维度 | 概念 A：**参数量 (Parameters)** | 概念 B：**计算量 (Operations/FLOPs)** | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **复杂度度量** | **计数对象** | 学习过程中需要更新的权重和偏置总数。由卷积核大小、内外通道数决定 [8, 16]。 | 运行一次前向传播所需的乘法和加法次数。与特征图的分辨率及步长密切相关 [17]。 | [8, 16, 17] |
| | **优势** | 卷积层由于**参数共享**，参数量远小于全连接层 [2, 18]。 | 虽然参数少，但由于要在全图滑动，计算量依然可能很大 [19]。 | [2, 18, 19] |

| 概念组 | 维度 | 概念 A：**填充 (Padding)** | 概念 B：**步长 (Stride)** | 来源 |
| :--- | :--- | :--- | :--- | :--- |
| **尺寸控制** | **主要作用** | 在输入边缘补零。用于**保持输出尺寸**及保护边缘像素信息 [5, 20]。 | 卷积核滑动的步距。主要用于**降维（下采样）**和减少冗余计算 [20, 21]。 | [5, 20, 21] |
| | **设计惯例** | 卷积层通常使用 Padding 以避免特征图快速萎缩 [2, 22]。 | 池化层步长通常等于核大小（如 $2\times2$ 核步长为 2）实现无重叠降维 [21]。 | [2, 21, 22] |
